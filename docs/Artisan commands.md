### 1. Генерация кода (Make)

Самая используемая группа команд. Вместо создания файлов вручную, используйте `make`.

- **`php artisan make:model Name`** Создает модель.
    - **Полезный лайфхак:** Используйте флаги, чтобы создать всё сразу: `php artisan make:model Post -a` (создаст Модель, Миграцию, Контроллер, Фабрику, Сидер и Политики). `php artisan make:model Post -mcr` (Модель + Миграция + Контроллер с методами ресурса).
- **`php artisan make:controller NameController`** Создает пустой контроллер.
    - `--resource`: создает контроллер с готовыми методами CRUD (index, create, store, etc.).
    - `--api`: то же самое, но без методов `create` и `edit` (для API).
- **`php artisan make:migration create_users_table`** Создает файл миграции.
- **`php artisan make:request StoreUserRequest`** Создает Form Request (валидация). Очень рекомендую использовать для валидации входящих данных вместо валидации прямо в контроллере.
- **`php artisan make:seeder UserSeeder`** Создает сидер (для заполнения БД тестовыми данными).
- **`php artisan make:factory PostFactory`** Создает фабрику (генератор фейковых данных для моделей).
- **`php artisan make:component Alert`** Создает компонент Blade (View).

### 2. Работа с Базой Данных

Эти команды управляют структурой и наполнением вашей БД.

- **`php artisan migrate`** Запускает все миграции (применяет изменения к БД).
- **`php artisan migrate:rollback`** Отменяет последнюю партию миграций (полезно, если ошиблись при создании таблицы).
- **`php artisan migrate:fresh`** **Внимание:** Удаляет все таблицы и запускает миграции заново.
    - `--seed`: Удаляет всё, создает заново и сразу заполняет тестовыми данными (`migrate:fresh --seed`). Это "золотая команда" для локальной разработки.

- **`php artisan db:seed`** Запускает сидеры (наполняет базу данными).

### 3. Очистка кэша и Отладка (Важно!)

Если вы поменяли код/конфиг, а изменения не видны — выполняйте эти команды.

- **`php artisan optimize:clear`** Универсальная команда. Очищает кэш конфигов, роутов, view и событий. Используйте её первой при странных багах.

- **`php artisan route:list`** Показывает список всех зарегистрированных маршрутов в приложении. Незаменимо для отладки ошибок "404" или проверки имен роутов.

- **`php artisan config:clear`** Очищает кэш конфигурации.
    - _Совет:_ В разработке (local) лучше вообще не кэшировать конфиг.

- **`php artisan storage:link`** Создает символическую ссылку из `public/storage` в `storage/app/public`. Обязательно для отображения загруженных картинок.

### 4. Запуск и Утилиты

- **`php artisan serve`** Запускает встроенный локальный сервер разработки (обычно на [http://127.0.0.1:8000](https://www.google.com/search?q=http://127.0.0.1:8000)).
- **`php artisan tinker`** Запускает интерактивную консоль (REPL). Здесь можно писать PHP-код и взаимодействовать с вашим приложением "на лету" (например, `User::find(1)`), чтобы проверить данные без создания контроллеров.
- **`php artisan key:generate`** Генерирует ключ приложения (`APP_KEY` в `.env`). Требуется один раз при установке проекта.

---

### 5. Очереди (Queue)

Если вы используете фоновые задачи (отправка писем, обработка видео).
- **`php artisan queue:work`** Запускает воркер, который обрабатывает задачи из очереди.
- **`php artisan queue:listen`** Похоже на `work`, но перезагружает код при каждом изменении (полезно при разработке, но медленнее).
