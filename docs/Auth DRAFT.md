## Аутентификация и авторизация в Laravel + React проекте (Draft)
черновик:
использовать Laravel Sanctum
Поскольку и публичная часть, и админка будут находиться на одном домене, вы можете использовать общую сессию на cookies


- **Public (SSR):** Laravel Blade (стандартный рендеринг на сервере) для SEO и скорости первой отрисовки.
- **Admin (CSR):** React (Client-Side Rendering) как SPA (Single Page Application) для динамики и удобства управления.

- **Backend:** Laravel.
- **Auth:** Laravel Sanctum (используем cookie-based сессии).
- **Public:** Обычные Blade шаблоны.
- **Admin:** React + React Router.
- **Вход в систему:**
    - Сделайте страницу логина на **Blade**. Это проще всего. Админ логинится через обычную форму, Laravel ставит куку и редиректит на `/admin`, где инициализируется React. React сразу может делать запросы к API, так как кука уже есть.

## Организация аутентификации (Login)

Здесь есть нюанс: React-админке нужно как-то авторизоваться. У вас есть два пути:
#### Вариант А: Laravel Fortify (Рекомендуемый для кастома)

Это backend-only пакет для аутентификации. Он не дает никаких View, но дает готовые энпоинты для логина.
1. Вы ставите Fortify.
2. Из React отправляете POST запрос на `/login`.
3. Fortify сам проверит пароль, создаст сессию и вернет 200 OK.
4. Sanctum «подхватит» сессию для дальнейших запросов к API.

#### Вариант Б: Общая страница входа (Blade)

Самый простой вариант.

1. Страница `/login` — это обычный Blade-шаблон (SSR).
2. Админ вводит логин/пароль -> Laravel редиректит его.
    - Если роль `user` -> на главную.
    - Если роль `admin` -> на `/admin` (где загружается React).
3. В React приложении вам вообще не нужно писать форму логина.



После того как аутентификация настроена, доступ к страницам ограничивается через **Middleware**
